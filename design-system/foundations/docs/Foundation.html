Foundation v2 · Design System

Warm Analytical Minimalism

A reusable foundation for premium, productive web applications. Warm-neutral surfaces, an olive primary, restrained terracotta accent, and operational density. Tokens are the contract; components reference semantic and component-level names — never primitives.

01

Principles

The seven rules that govern every decision in this system. When in doubt, go with the principle, not the example.

01 / Information before ornament

Decoration must improve comprehension or be removed. Borders, weight and rhythm carry hierarchy — not gradients.

02 / One primary, one accent

Olive for action. Terracotta for emphasis. Blue-grey for information. Status colors carry status. No new accents.

03 / Hierarchy via weight + leading

Reach for weight before pixels. Two visible weights and three sizes carry an entire screen.

04 / Calm motion

Sub-200ms with gentle easing. Motion confirms a change, never performs one. Honour reduced-motion.

05 / Density is a setting

compact · cozy · comfortable. Operational surfaces default to compact; reading surfaces to comfortable.

06 / Borders before shadows

Elevation only when surfaces genuinely float — popovers, modals, toasts. Cards are flat by default.

07 / Tokens are the contract

Application code references semantic and component tokens. Primitives are private. Theming is one file away.

05b

Charts & data visualization

A complete chart palette: an 8-color categorical set, a sequential ramp, a diverging ramp, and semantic accents for thresholds, targets, and reference lines. Framing tokens (grid / axis / tooltip / crosshair) live alongside the data colors so axes never compete with marks.

Categorical · 8 series

Use 1–3 series for most product charts. Beyond 4, sort by value and let the lower series share a muted tone.

Sequential · single-hue

Density, intensity, magnitude.

Diverging · negative ↔ positive

Variance, P&L, sentiment, deltas vs. target.

Semantic accents
TokenUse
--color-chart-positiveUp movements, gains, success states
--color-chart-negativeDown movements, losses, failures
--color-chart-targetGoal line, planned value, benchmark target
--color-chart-thresholdWarning band, SLA limit
--color-chart-referenceMean, median, baseline overlay
--color-chart-projectionForecast / extrapolation segment
--color-chart-anomalyOutliers, flagged points
Choropleth · map ramps

Map regions consume --map-seq-* (single-hue intensity) or --map-div-* (negative-to-positive). Boundaries, labels, hover, selection, and no-data states have their own tokens — they remain legible on either ramp without recoloring.

Five sizes, three stroke weights, and a small set of semantic colors. Icons inherit currentColor by default so they pick up text color automatically; use the dedicated --color-icon-* tokens only when an icon's color must be decoupled from surrounding text.

08

Accessibility

Body text exceeds WCAG AA at 14px. Status text is always rendered with the -text token (700-level), never -icon or raw -500. Focus is always visible via :focus-visible.

text-secondarybackground-page~7.9 : 1Pass · AAAPass · AAA
text-mutedbackground-page~5.2 : 1Pass · AAPass · AAA
action-primary-fgaction-primary~5.6 : 1Pass · AAPass · AAA
status-success-textstatus-success-bg~5.4 : 1Pass · AAPass · AAA
status-warning-textstatus-warning-bg~5.1 : 1Pass · AAPass · AAA
status-error-textstatus-error-bg~5.7 : 1Pass · AAPass · AAA

Status badges rely on color + a dot + text label. Form errors include a red border, an error message, and an icon. Charts include legends and patterning options for colorblind users.

09

Architecture

Three layers, one direction of reference: components → semantic → primitives. Product code never reaches past the semantic boundary.

/design-system
  /tokens
    primitives.css        raw OKLCH ramps, sizing, motion (private)
    semantic.css          --color-{category}-{property}-{modifier}
    components.css        --{component}-{property}-{state}
    shadcn.css            shadcn/ui bridge (--accent → neutral hover)
    tokens.json           W3C $type/$value/$description tree
    tailwind.tokens.ts    theme.extend export
  /styles
    base.css              resets, focus, motion preferences
    themes.css            theme activation strategy
  /docs
    Foundation.html       this living spec
    DESIGN_SYSTEM.md      principles + rules
    IMPLEMENTATION.md     React/Next.js/Tailwind/shadcn setup

10

Self-evaluation

Honest scorecard against the goals set at kickoff. Where the system is tight, where it's deliberately scoped, and what's reasonable to add next.

GoalScoreNotes
Distinct visual identity
Avoid generic SaaS · feel premium & calm		 strong Warm-neutral foundation + olive primary + restrained terracotta is a defensible position. Editorial Newsreader on otherwise utilitarian Plex Sans gives one signature moment without theatrics.
Token architecture
Three layers · one contract		 strong primitives → semantic → components is enforced. Naming is normalised to --color-{category}-{property}-{modifier}. Status uses four-token quartets. Tailwind, shadcn, and W3C JSON all bind to the same source of truth.
Light + dark parity strong Dark mode is a semantic remap only — primitives never change. Shadows weaken; borders carry separation. Status quartets re-bind to 200/700 pairings on dark surfaces (WCAG AAA).
Accessibility strong All text-on-surface pairings clear AA at 14px; primary text reaches AAA. Status text always uses the -text token. Reduced-motion honoured. Color is never the only signal.
Component coverage strong 30+ surfaces tokenised: button, input, textarea, select, checkbox, radio, toggle, card, badge, alert, table, metric, sidebar, topbar, nav-item, tabs, segmented, breadcrumb, stepper, pagination, progress, skeleton, spinner, dropdown, popover, tooltip, modal, command palette, toast, empty state, file upload, date picker. Each consumes a --{component}-* contract bound to semantic tokens — light + dark out of the box.
Charting scoped Eight categorical hues + sequential ramp + framing tokens (grid/axis/tooltip) + semantic positive/negative/threshold. No documented choropleth or diverging ramp yet — add when first needed.
Theming strong data-theme="dark" for mode, data-theme-brand="…" for product re-skin via semantic overrides. Primitives are never overridden — guarantees palette integrity.
Icon system strong Five sizes (14/16/20/24/32 px) and three stroke weights as primitives, plus semantic icon colors (primary / secondary / muted / disabled / inverse / accent + status quartet links) with full dark-mode bindings. Component-scoped sizes (--input-icon-size, --badge-icon-size, etc.) keep usage consistent. Pair with Lucide rendered via currentColor.
Documentation strong Three Markdown files (DESIGN_SYSTEM, IMPLEMENTATION, ACCESSIBILITY) plus this Foundation page. Rules are written as rules, not guidelines — the system is opinionated by design.

(3) Contrast unit tests in CI (already speced in IMPLEMENTATION §10). (4) Diverging chart ramp + map/choropleth tokens when first product needs them. (5) Codemod for v1→v2 token renames.
