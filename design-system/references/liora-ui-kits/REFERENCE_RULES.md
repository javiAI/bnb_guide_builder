# Reference Rules

## Allowed

- Use HTML files as visual references.
- Extract layout patterns.
- Extract component composition.
- Extract information hierarchy.
- Compare operator, messaging and guest surfaces.

## Not allowed

- Do not copy static HTML directly into React routes.
- Do not import `colors_and_type.css` into the app.
- Do not use legacy reference tokens in production.
- Do not replace Product Design Foundations tokens.
- Do not treat this folder as a second design system.

## Selected references

- Operator overview: `operator/index.html`
- Operator subpages: `operator/subpages.html`
- Messaging: `messaging/index.html`
- Guest: `guest/index.html`
